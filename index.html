The 3AM Incident — A Crash Course in Observability
incident · production · hard lesson
● incident ›

What happened

A production endpoint silently started failing. No alerts fired. No meaningful logs existed. PM2 was restarting the process on crash — masking the root cause entirely.

Root cause

Unhandled promise rejections in an async Express route handler. Node.js swallowed the error, the process crashed, PM2 restarted it. The crash-restart loop caused data inconsistencies.

What I changed

Global unhandled rejection handlers, structured JSON logging, health-check endpoints on every service. These now go in before any feature code — non-negotiable.

The real lesson

You don't know your system is broken until you have observability. A system with no logs isn't a running system — it's a system that hasn't failed visibly yet.

→ Reliability is a first-class feature, not post-launch polish.
→ PM2 process restarts are not error handling — they're a silent cover-up.
→ Structured logs and health endpoints ship on day one, before features.
